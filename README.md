# МИС: Модуль трекера задач

Сервис для управления задачами медицинского персонала с поддержкой периодических (регулярных) задач.
Решение тестового задания на позицию Junior Backend Developer.

## Описание проекта

Проект представляет собой REST API для управления задачами врачей (создание, редактирование, удаление, просмотр). Добавлена новая функциональность: возможность задавать настройки периодичности задачи. Теперь задачи могут создаваться автоматически на основе заданных параметров.

Кроме того, был реализован **UI-интерфейс** (Frontend), полностью повторяющий концептуальный дизайн из ТЗ. Интерфейс раздается самим Go-сервером.

### Принятые технические решения и допущения:
1. **Шаблоны и Инстансы**: Задача с заданными параметрами повторяемости (recurrence) становится "шаблоном". Воркер (горутина), работающий в фоне раз в минуту, находит шаблоны, для которых наступило время генерации (`next_generate_date`), и создает "инстансы" — реальные разовые задачи с ссылкой на родительский шаблон (`parent_task_id`).
2. **Типы периодичности**:
   - `daily` (Ежедневно / каждые N дней)
   - `monthly` (В определенные числа месяца, например 1 и 15 числа)
   - `specific_dates` (Только в указанные даты)
   - `even_days` / `odd_days` (По четным и нечетным числам)
3. **Хранение данных**: PostgreSQL. Схема базы дополнена новыми полями для поддержки рекуррентности.
4. **Архитектура**: Clean Architecture (Domain, Usecase, Repository, Transport HTTP). Разделение на логические слои, строгая типизация, валидация DTO.
5. **UI**: Написан на чистом HTML/JS с использованием TailwindCSS для быстрого и красивого результата, чтобы продемонстрировать Fullstack-возможности.

## Быстрый запуск (Docker)

Проект полностью докеризован и готов к локальному запуску одной командой.

1. Скопируйте пример переменных окружения:
   ```bash
   cp .env.example .env
   ```

2. Запустите контейнеры:
   ```bash
   docker-compose up --build -d
   ```

3. Откройте в браузере:
   - **Frontend UI**: [http://localhost:8080/](http://localhost:8080/)
   - **Swagger Документация**: [http://localhost:8080/swagger/](http://localhost:8080/swagger/)

> **Важно:** Если база данных уже запускалась со старой схемой, необходимо очистить volume для применения новых миграций:
> `docker-compose down -v && docker-compose up --build -d`

## Примеры запросов к API (cURL)

**1. Создание разовой задачи (без повторений)**
```bash
curl -X POST http://localhost:8080/api/v1/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Осмотреть пациента Иванова",
    "description": "Проверить послеоперационные швы",
    "status": "new",
    "scheduled_at": "2026-10-25T10:00:00Z"
  }'
```

**2. Создание ежедневной задачи (каждые 2 дня)**
```bash
curl -X POST http://localhost:8080/api/v1/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Обзвон пациентов",
    "description": "Уточнить самочувствие",
    "status": "new",
    "recurrence_type": "daily",
    "recurrence_interval": 2
  }'
```

**3. Создание ежемесячной задачи (1 и 15 числа)**
```bash
curl -X POST http://localhost:8080/api/v1/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Отчетность",
    "description": "Сформировать отчет по отделению",
    "status": "new",
    "recurrence_type": "monthly",
    "recurrence_month_days": [1, 15]
  }'
```

**4. Получить список всех задач**
```bash
curl -X GET http://localhost:8080/api/v1/tasks
```
